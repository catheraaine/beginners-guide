Dotfiles

Summary

What & Why

A dotfile is any file that starts with a dot. They are where all of your configurations for your computer are stored. Sometimes your machine may come with dotfiles, sometimes you will have to create the files yourself. They are usually in a shell command language. My examples will be for the Bash shell language.

Aliases

Aliases are commands assigned to custom shortcuts. They follow a very specific syntax alias shortcut="command". You can make one directly in the command line, but to create a permanent alias, put it in an alias dotfile, .aliases, and link that to your main dotfile. This is in my .bashrc and links my other dotfiles, since .bashrc, is the file the computer runs. for file in ~/.{path,bash_prompt,aliases,functions}; do [ -r "$file" ] && [ -f "$file" ] && source "$file"; done; unset file;

Functions

Functions are similar to aliases. You set them in a dotfile, and they consist of creating shortcuts to run a command. Yet, functions can be more complex than aliases, and perform multple actions. They have a similar syntax to functions in JS. function name() { commands }.

Cheat Sheet

Aliases

alias c="clear"
alias ll="ls -lha"
alias rbash="source ~/.bashrc"
alias gs="git status"
alias ga="git add"
alias gc="git commit -m"

Functions

# make a directory and go to it function mkd () { mkdir -p "$@" && cd "$@" }
# go to the directory and list out the contents function cl () { cd $@ && ls -la }
# congratulate yourself with a happy dance function happydance { echo ' ┏(-_-)┛ ┗(-_-﻿ )┓ ┗(-_-)┛ ┏(-_-)┓ ' }

Projects to Complete

Create your own dotfiles

Check out some of the example resources, and start creating your own aliases and functions. Start small, add new ones as you work on projects, and don't be afraid to be creative. If using bash, you will want a .bashrc, .bash_profile, .bash_prompt, .aliases, .functions

Create a dotfiles repo

Create a repo on GitHub or any other similar service, and add your dotfiles to the repo. Then you will be able to pull them down for any new system you're on, and you can practice commits.
